# Archive: uv-ruff-mypy-install

## The request

The project used Poetry for dependency management and flake8, black, isort, pydocstyle, and mypy for linting/formatting. The feature requested:

1. **Migrate to uv** — Use uv for package management; move project metadata to PEP 621 `[project]` and optional PEP 735 `[dependency-groups]`; replace `poetry.lock` with `uv.lock`.
2. **Linting and formatting** — Replace flake8, black, isort, and pydocstyle with **Ruff** for lint and format; keep **mypy** for type checking; centralise config in `pyproject.toml`.
3. **CI and pre-commit** — Update GitHub Actions to use uv and run Ruff and mypy (and pytest); update `.pre-commit-config.yaml` to Ruff (and optionally mypy) instead of black and flake8.
4. **install.sh** — Add a single script at repo root that: ensures uv is available (install via official installer if missing); removes `.venv`, `__pycache__`, `.pytest_cache`; creates a venv with uv and installs the project in editable mode including dev deps; runs pytest at the end. Reference: `tap-rest-api-msdk/install.sh`; run pytest directly, not tox.

**Testing:** Run `./install.sh` and confirm uv is used, env is clean, editable install with dev deps, and pytest passes; CI runs uv, ruff, mypy, pytest and passes; pre-commit runs Ruff (and optionally mypy) and passes; no regressions in existing tests or type checks.

---

## Planned approach

**Solution choice (from planning):** uv as package manager; `uvx migrate-to-uv` for initial Poetry → PEP 621 conversion, then manual tweaks for semantic_release and scripts. Ruff for lint and format (replacing flake8/black/isort/pydocstyle); mypy retained. install.sh uses the official uv installer when uv is missing; CI uses `astral-sh/setup-uv`; no tox — `uv run pytest` in CI and install.sh.

**Architecture:** Tooling and config only — no new application components. Single source of truth: `pyproject.toml` for metadata, dependency-groups, build-system (hatchling), Ruff, mypy, and semantic_release (`version_toml` → `project.version`, `build_command` → uv lock + uv build). New artifacts: `install.sh` (bootstrap + test), `uv.lock` (generated by `uv lock`). CI: checkout → setup-uv → Python → `uv sync` → ruff check → ruff format --check → mypy → pytest (and optional coverage); release job keeps semantic-release with uv build_command from pyproject. Pre-commit: Ruff repo (astral-sh/ruff) for check and format; black and flake8 removed; optional mypy hook.

**Implementation order (5 phases):** (1) Migrate pyproject.toml to PEP 621 + dependency-groups and add Ruff/mypy config and semantic_release updates; generate uv.lock and remove poetry.lock. (2) Create install.sh (ensure uv, clean, uv venv + uv sync, uv run pytest). (3) Add/update `.github/workflows/ci.yml` with uv, Ruff, mypy, pytest. (4) Update `.pre-commit-config.yaml` to Ruff (and optionally mypy). (5) Update README and any contributor docs for uv, Ruff, mypy, install.sh.

**Task breakdown (6 tasks):** 01 — Migrate pyproject.toml (PEP 621, dependency-groups, hatchling, Ruff, mypy, semantic_release); 02 — Generate uv.lock, remove poetry.lock, verify uv sync and pytest; 03 — Create install.sh; 04 — Add/update ci.yml; 05 — Update pre-commit to Ruff; 06 — Update README and docs. Validation at each step: tool runs (e.g. `uv lock`, `uv run pytest`) and black-box outcomes (exit codes, presence of `.venv`); no new application unit tests; no assertions on call counts or log lines.

## What was implemented

All six tasks were completed:

- **Task 01 (migrate pyproject.toml):** `pyproject.toml` converted to PEP 621 `[project]` and PEP 735 `[dependency-groups]` with a `dev` group; `[build-system]` set to hatchling; Poetry, black, and isort sections removed; `[tool.ruff]` (and lint/format) and `[tool.mypy]` added; `[tool.semantic_release]` updated to `version_toml = ["pyproject.toml:project.version"]` and a uv-based `build_command`. Validation: `uv lock` succeeds.

- **Task 02 (generate uv.lock):** `uv.lock` generated via `uv lock`; `poetry.lock` removed. Verified: `uv sync` and `uv run pytest` pass.

- **Task 03 (create install.sh):** `install.sh` added at repo root: ensures uv on PATH (official installer if missing), cleans `.venv`/`__pycache__`/`.pytest_cache`, runs `uv venv` and `uv sync`, then `uv run pytest` (script exit code = pytest exit code). Script is executable and commented per phase.

- **Task 04 (update CI):** `.github/workflows/ci.yml` added/updated with: checkout, astral-sh/setup-uv, Python setup, `uv sync`, then `uv run ruff check .`, `uv run ruff format --check .`, `uv run mypy target_gcs`, `uv run pytest` (and coverage). Release job uses semantic-release with uv on PATH.

- **Task 05 (update pre-commit):** `.pre-commit-config.yaml` updated to use Ruff (ruff-check, ruff-format); black and flake8 removed.

- **Task 06 (update documentation):** README Developer Resources (and any related docs) updated for uv, install.sh, Ruff, mypy, and pytest; Poetry and old linter references removed.

**Outcomes:** Single-command bootstrap and test via `./install.sh`; reproducible installs via `uv.lock` and `uv sync`; CI and pre-commit run the same toolchain (uv, Ruff, mypy, pytest). No application behaviour changes; existing tests and type checks remain valid under the new tooling.
